Make agent skills reliable.

Start with Doctor. Install SkillSpec. Make it your default way to create and run skills that agents can actually follow.

SkillSpec is an open-source CLI and contract format for SKILL.md packages. It checks a skill, turns it into a routeable workflow, guides execution through phases, tools, and checks, then records proof of what actually happened.

Entry point

Try Doctor on one public skill URL.

Doctor is the fastest way to see the reliability gap. Paste a public GitHub skill, plugin, or repo URL and get a report that shows where agent execution may drift.

Folder URLs using /tree/<branch>/... are canonical. Copied /blob/<branch>/... URLs also work when the path is a folder. This opens a public GitHub issue, so do not submit private repositories, tokens, credentials, or sensitive internal paths.

Install next

Make SkillSpec your default skill path.

Use the public Doctor for a first look, then install the CLI locally. From there, create, convert, route, and run skills with the same verifier-executor loop you can publish with your project. 

curl -fsSL https://skillspec.sh/install.sh | sh
skillspec doctor ./skills/my-skill
/skillspec import ./skills/my-skill
/skillspec install router

Who it is for

One contract for users, authors, and product teams shipping skills.

Skill users

Run installed skills with precise routes, allowed tools, required checks, and proof instead of hoping the agent remembers every step.

Skill authors

Publish skills that explain when they should run, how they should run, and what evidence proves they completed correctly.

Tool authors

Ship product skills that guide agents through the right workflow, reduce misuse, and make advanced features easier to adopt.

Hardware and software teams

Package support, setup, diagnostics, and automation skills so customers get repeatable action instead of fragile prompt recipes.

Adoption path

Try it once. Then make it how you create and use skills.

1

Check the skill you already have

Run Doctor on a public URL or a local folder. The report shows where execution can skip, drift, overload context, miss dependencies, or finish without proof. 

# Public GitHub URI
skillspec doctor \
  https://github.com/org/repo/tree/main/skills/example \
  --markdown > docs/skillspec-doctor-baseline.md

# Local folder
skillspec doctor ./skills/example \
  --markdown > docs/skillspec-doctor-baseline.md
2

Install the verifier-executor

Install the CLI so SkillSpec can check private skills, compile contracts, refresh the router, and guide real work from your local checkout. 

curl -fsSL https://skillspec.sh/install.sh | sh
skillspec --help
3

Create or convert the skill

Convert important prose into a contract that names the route, phases, tool boundaries, checks, and completion proof the agent should follow. 

# Existing skill
/skillspec import ./skills/my-skill

# New product skill
Write SKILL.md, then import and prove it.
4

Run through the router by default

Use router mode so skills are selected intentionally, executed through the SkillSpec path, and checked before they become the agent's first answer.

The visible loader stays small while SkillSpec handles routing, phase order, checks, resume, and proof.

5

Ship proof with the skill

Publish the Doctor baseline, generated skill.spec.yml, compiled loader, and alignment report so users can see exactly how the skill is meant to run.

This is the default path for companies shipping agent skills with hardware, software, APIs, or internal operations.
